# Secure File Sharing System

A secure client–server file transfer system that ensures **confidentiality, integrity, and controlled access** using modern cryptographic techniques.

The system encrypts files using **AES-256-GCM**, protects encryption keys with **RSA-2048**, verifies integrity with **SHA-512**, and restricts access using **API key authentication**.

It also includes a command-line client, an administrative key management system, and a comprehensive test suite covering cryptographic modules and full end-to-end file transfers.

> **Requires Python 3.10 or higher**

---

# Architecture Overview

The system follows a **secure hybrid encryption architecture**.

**Upload flow:**
```
Client uploads file
→ Server encrypts file with AES-256-GCM
→ AES key encrypted using RSA-2048
→ Encrypted file + encrypted key stored on disk
```

**Download flow:**
```
Client requests file
→ Server decrypts AES key with RSA private key
→ File streamed and decrypted chunk by chunk
→ Client receives original file
```

# Security Features

## Hybrid Encryption

Each uploaded file is encrypted using **AES-256-GCM authenticated encryption**, which provides both confidentiality and tamper detection.

## RSA Key Protection

The AES key used to encrypt the file is itself encrypted using **RSA-2048 OAEP** before being stored.

This prevents exposure of symmetric keys even if storage is compromised.

## Integrity Verification

Files are hashed using **SHA-512** to ensure that the uploaded and downloaded data remain identical.

The client calculates a local hash and verifies it against the server response.

## API Key Authentication

Every protected endpoint requires an API key.

API keys are:
- Generated by the admin
- Stored using **salted Scrypt hashing**
- Time-limited (10 days)
- Revocable at any time

## Secure API Key Storage (Client)

The client encrypts its API key locally using:
- AES-GCM encryption
- Scrypt-derived key from a user passphrase

This prevents API keys from being stored in plaintext.

## Rate Limiting

The server protects against abuse using request rate limits on all endpoints.

## Audit Logging

All critical events such as uploads, downloads, and key rotations are recorded in a rotating audit log.

## UUID File Identification

Files are referenced using UUID identifiers to prevent predictable file enumeration.

# Installation and Setup

## 1. Clone the Repository
```bash
git clone https://github.com/RB1507/secure-file-sharing.git
cd secure-file-sharing
```

## 2. Install Dependencies
```bash
pip install -r requirements.txt
```

## 3. Configure Environment
```bash
cp .env.example .env
```

Open `.env` and fill in your values:
```
ADMIN_MASTER_KEY=your-strong-admin-key-here
RSA_KEY_PASSWORD=your-strong-rsa-password-here
FLASK_ENV=production
```

## 4. Start the Server
```bash
python app.py
```

The server starts on `http://0.0.0.0:5000` by default.

## 5. Generate an API Key (Admin)
```bash
python client.py admin gen-key
```

This prints an API key to share with users.

## 6. Setup the Client (Each User — First Time Only)
```bash
python client.py setup
```

You will be asked for:
- Server address (e.g. `http://192.168.1.10:5000`)
- API key (provided by admin)
- A passphrase to protect the key locally

# Server Features

The Flask server provides the secure file storage backend.

Key capabilities include:
- Secure file upload and download
- Streaming encryption for large files
- AES-256-GCM chunk-by-chunk encryption
- RSA-2048 key wrapping
- API key authentication with Scrypt hashing
- Admin key rotation and revocation
- UUID-based file identification
- Rotating audit logging
- Rate limiting on all endpoints
- Metadata storage (filename, size, hash)

---

# Client Features

The CLI client allows users to securely interact with the server.

Capabilities include:
- Upload files securely
- Download files using file ID
- Encrypted local API key storage
- Session caching (passphrase asked once per 8 hours)
- Automatic integrity verification on upload
- Simple one-time setup workflow
- Admin key management commands

---

# Cryptographic Modules

## AES Module (`encryption/aes.py`)
Provides AES-256-GCM encryption and decryption with authentication tag verification and secure random nonce generation.

## RSA Module (`encryption/rsa.py`)
Handles RSA-2048 key generation, password-protected key storage, AES key encryption and decryption using OAEP padding.

## Hash Module (`encryption/hash.py`)
Implements SHA-512 hashing for verifying file integrity before and after transfer.

# Security Model

| Principle | Implementation |
|---|---|
| Confidentiality | AES-256-GCM file encryption |
| Integrity | SHA-512 hash verification |
| Authentication | Scrypt-hashed API keys |
| Key protection | RSA-2048 wraps AES keys |
| Tamper detection | AES-GCM authentication tag |
| Client key safety | Passphrase-encrypted local keystore |
| Abuse prevention | Rate limiting on all endpoints |
| Audit trail | Rotating audit log for all events |
